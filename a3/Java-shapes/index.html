Assignment 3

Introduction

You will use inheritance from abstract classes and interfaces in Java to describe regular shapes (circle, solid, square) that are to be placed on a canvas (some abstract graphics context). You will also start using Javadoc-style comments.

To do

Find the following files in ~kschmidt/public_html/CS265/Assignments/Java-shapes :

  • RegularShape.java - an abstract base class for shapes, describes behaviors we'd expect of shapes. No modifications are req'd
  • GuiObj.java - an interface, describes behaviors for any closed object that would be placed on a canvas, colored, and moved. No modifications are req'd
  • GuiRegShape.java - A subclass of RegularShape that implements the GuiObj i/f. All of the interface can be implemented here. I've done a little. You need to fill in the rest. Search for TODO w/in the file

You will provide the following classes, subclasses of GuiRegShape:

  • Circle
  • Triangle
  • Square

Each class will be in its own file, and will supply behavior to the previously abstract (undefined) methods. Also, provide 2 constructors, as the GuiRegShape class does. Each file will have Javadoc-formatted comments for the class, and for each method and c'tor. Please use the other files as a template.

Finally, you will supply your test driver, that will be filled in as you implement each class, above. To test the virtual methods your code would look something like this: 

GuiRegShape s = new Square( 3, "square1" ) ;
s.move( 3, 7 ) ;
s.color( Color.ORANGE ) ;
...

The test driver will be in a class called Prog . You should test all of the methods, for each object.

Please be careful when checking float types for equality. That is, don't. Subtract, look for a very small difference (hint: a little helper method here might be a good idea).
